<objective>
Adapt GridView for mobile with responsive column sizing, smaller icons, and adjusted padding.

Purpose: The GridView uses `minmax(140px, 1fr)` which may be too wide for small phones (375px viewport), potentially showing only 2 items per row with excessive spacing. Reducing the minimum to 100px and adjusting icon/padding sizes creates a denser, more usable mobile grid.
Output: Updated explorer.css with mobile GridView rules. CSS-only solution.
</objective>

<task type="auto">
  <name>Task 1: Add mobile GridView CSS rules</name>
  <files>frontend/src/styles/explorer.css</files>
  <action>
  Inside the existing `@media (max-width: 768px)` block in explorer.css, add GridView-specific mobile styles:

  ```css
      /* ===== GridView Mobile ===== */
      /* Reduce minimum column width for tighter grid on small screens */
      .grid-view {
          grid-template-columns: repeat(auto-fill, minmax(100px, 1fr));
          gap: var(--spacing-sm);
      }

      /* Grid items: reduce padding for mobile */
      .grid-item {
          padding: var(--spacing-sm);
      }

      /* Grid item icon: smaller on mobile */
      .grid-item-icon {
          width: 48px;
          height: 48px;
          margin-bottom: var(--spacing-xs);
      }

      .grid-item-icon svg {
          width: 36px;
          height: 36px;
      }

      /* Grid item label: smaller font on mobile */
      .grid-item-label {
          font-size: 12px;
      }

      /* Grid item metadata: smaller on mobile */
      .grid-item-meta {
          font-size: 10px;
      }

      /* KG LED indicator: adjust position for smaller grid items */
      .kg-ready-led {
          top: 4px;
          right: 4px;
          width: 8px;
          height: 8px;
      }
  ```

  This creates a more compact grid for mobile:
  - `minmax(100px, 1fr)` allows 3 columns on a 375px screen (vs 2 at 140px)
  - Smaller icons (48x48 → 36x36 svg) and less padding
  - Smaller font sizes for labels and metadata
  - KG LED repositioned for smaller items

  WHY 100px minimum: On iPhone SE (375px), with 8px gaps and 8px padding on .explorer-content, the available width is ~359px. At 100px min: 3 columns × 100px = 300px + 2 gaps = 316px, fitting perfectly. At 140px min: only 2 columns fit with excessive whitespace.
  </action>
  <verify>Run `cd frontend && npm run build` to verify build succeeds</verify>
  <done>GridView shows 3 columns on mobile, compact icons/labels, build passes</done>
</task>

  This handles iPhone SE and similar very small screens (320-375px) with further compaction.
