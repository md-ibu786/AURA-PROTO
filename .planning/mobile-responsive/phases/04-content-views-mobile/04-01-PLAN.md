<objective>
Adapt ListView for mobile by hiding auxiliary columns and simplifying the grid template.

Purpose: The ListView currently uses a 5-column grid (`40px 1fr 120px 100px 100px`) which causes horizontal scroll on a 375px mobile screen. By hiding Type, Modified, and Count columns on mobile, the list becomes a clean 2-column layout (icon + name) that fits any screen width.
Output: Updated explorer.css with mobile ListView rules. No React component changes needed (CSS-only solution).
</objective>

<task type="auto">
  <name>Task 1: Add mobile ListView CSS rules</name>
  <files>frontend/src/styles/explorer.css</files>
  <action>
  Inside the existing `@media (max-width: 768px)` block in explorer.css, add ListView-specific mobile styles:

  ```css
      /* ===== ListView Mobile ===== */
      /* Simplify to 2-column grid: icon + name only */
      .list-header,
      .list-row {
          grid-template-columns: 32px 1fr;
          gap: var(--spacing-sm);
          padding: var(--spacing-sm);
      }

      /* Hide auxiliary columns: type, date, count */
      .list-header > *:nth-child(n+3),
      .list-row > *:nth-child(n+3) {
          display: none;
      }

      /* Ensure list row has adequate touch target height */
      .list-row {
          min-height: 44px;
          padding: var(--spacing-sm) var(--spacing-md);
      }

      /* List row icon: slightly smaller on mobile */
      .list-row-icon {
          width: 20px;
          height: 20px;
      }

      /* List row name: ensure text doesn't overflow */
      .list-row-name {
          font-size: 14px;
          min-width: 0;
      }
  ```

  This is a pure CSS solution:
  - Uses `nth-child(n+3)` to hide the 3rd, 4th, and 5th columns (Type, Modified, Count)
  - Reduces grid to `32px 1fr` (icon + name)
  - Ensures 44px minimum touch target height
  - Works without any React component modifications

  WHY CSS-only: The ListView component renders all columns via grid children. Using CSS display:none is cleaner than conditional rendering because it avoids modifying component logic and keeps the responsive behavior purely in the stylesheet where all other responsive rules live.
  </action>

  <done>ListView shows only icon + name on mobile, no horizontal scroll, 44px touch targets, build passes</done>
