<task type="auto">
  <name>Task 1: Add mobile base styles to explorer.css</name>
  <files>frontend/src/styles/explorer.css</files>
  <action>
  Add a new section at the END of `explorer.css` (after the Aura Mode Toggle section, line ~1228). Add the following CSS:

  ```css
  /* ===== Mobile Responsive ===== */
  @media (max-width: 768px) {
      /* Layout: keep row direction but hide sidebar by default */
      .explorer-layout {
          position: relative;
          overflow-x: hidden;
      }

      /* Sidebar: hidden off-screen by default, slides in as overlay */
      .explorer-sidebar {
          position: fixed;
          top: 0;
          left: 0;
          bottom: 0;
          width: 280px;
          z-index: 100;
          transform: translateX(-100%);
          transition: transform 0.3s cubic-bezier(0.4, 0, 0.2, 1);
          box-shadow: none;
      }

      /* Sidebar open state */
      .explorer-sidebar.mobile-open {
          transform: translateX(0);
          box-shadow: 4px 0 24px rgba(0, 0, 0, 0.5);
      }

      /* Sidebar backdrop */
      .sidebar-backdrop {
          position: fixed;
          inset: 0;
          background: rgba(0, 0, 0, 0.5);
          z-index: 99;
          opacity: 0;
          pointer-events: none;
          transition: opacity 0.3s ease;
      }

      .sidebar-backdrop.visible {
          opacity: 1;
          pointer-events: auto;
      }

      /* Main content takes full width */
      .explorer-main {
          width: 100%;
          min-width: 0;
      }

      /* Content area: reduce padding on mobile */
      .explorer-content {
          padding: var(--spacing-sm);
      }
  }
  ```

  This creates:
  1. The sidebar as a fixed overlay that slides in from the left
  2. A backdrop element (the React component will render this)
  3. The main content taking full width
  4. Reduced padding in the content area

  Do NOT modify any existing desktop styles. Only ADD the new media query block at the end.
  </action>
  <verify>Run `cd frontend && npm run build` to verify CSS is valid and build succeeds</verify>
  <done>Mobile media query block exists at end of explorer.css, build passes, no desktop styles affected</done>
</task>

<task type="auto">
  <name>Task 2: Add overflow prevention to index.css</name>
  <files>frontend/src/styles/index.css</files>
  <action>
  Add a mobile media query at the END of `index.css` to prevent horizontal scrolling:

  ```css
  /* ===== Mobile Overflow Prevention ===== */
  @media (max-width: 768px) {
      html, body, #root {
          overflow-x: hidden;
          max-width: 100vw;
      }
  }
  ```

  This is a safety net that prevents any component from causing horizontal scroll on mobile. The `#root` selector matches the existing `#root` rule in index.css.

  Do NOT modify any existing styles. Only ADD the new block at the end of the file.
  </action>
  <verify>Run `cd frontend && npm run build` to verify CSS is valid and build succeeds</verify>
  <done>Overflow prevention rules added, build passes, no horizontal scroll possible on mobile</done>
</task>
