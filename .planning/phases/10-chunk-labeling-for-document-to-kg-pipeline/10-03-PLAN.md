# Plan 10-03: Frontend Types and Processing Stage

## Goal

Update frontend TypeScript types to include the new `chunk_labels` field from
the backend, and add a `LABELING` processing state to the Celery task progress
tracking so the frontend can display labeling progress during document
processing.

- [ ] `KGStatusResponse` interface includes `chunk_labels?: string[]` field
- [ ] `ProcessingState` enum includes `LABELING` value
- [ ] `update_progress` method maps labeling progress correctly
- [ ] Frontend build succeeds (`npm run build` in `frontend/`)

**Change 2:** Update the `update_progress` method (lines 262-292)
to include the LABELING state in the progress-to-state mapping.
The current if/elif chain maps progress ranges to states. Insert
`LABELING` between `CHUNKING` (10-25%) and `EMBEDDING` (30-50%).
Adjust the boundaries:

```python
    def update_progress(
        self,
        stage: str,
        progress: int,
        meta: Dict | None = None,
    ):
        """Update task progress state.

        Args:
            stage: Current processing stage name
            progress: Progress percentage (0-100)
            meta: Additional metadata to include
        """
        state_meta = {
            "stage": stage,
            "progress": progress,
            "started_at": getattr(
                self, "_start_time", None
            ),
        }
        if meta:
            state_meta.update(meta)

        self.update_state(
            state=ProcessingState.PARSING.value
            if progress < 10
            else ProcessingState.CHUNKING.value
            if progress < 25
            else ProcessingState.LABELING.value
            if progress < 30
            else ProcessingState.EMBEDDING.value
            if progress < 50
            else ProcessingState.EXTRACTING.value
            if progress < 70
            else ProcessingState.STORING.value
            if progress < 90
            else ProcessingState.COMPLETED.value,
            meta=state_meta,
        )
```

The key change is inserting the `LABELING` state mapping at the
25-30% progress range, and adjusting `CHUNKING` from `< 30` to `< 25`.
